# 答辩准备(PPT、讲稿、Q&A、评委调研)

答辩通常 10~15 分钟陈述 + 5~10 分钟提问。下面的做法出自实际答辩验证。

## 1. 答辩 PPT(约 20 页,对应 13~15 分钟)

**结构模板**(理工科,按学生课题映射):

| 页 | 内容 | 用时参考 |
|----|------|---------|
| 1 | 封面:题目、姓名学号、专业、指导教师 | 20s |
| 2 | 目录(选题背景/系统模型/问题建模/算法设计/仿真结果/总结展望) | 20s |
| 3-4 | 选题背景与意义:领域痛点 → 技术机遇 → 本课题要解决的问题(一页一个递进) | 1.5min |
| 5 | 研究内容总览:一张总框图,四块贡献 | 1min |
| 6-8 | 系统模型:场景图 + 核心建模(只放最关键的 2~4 条公式) | 2.5min |
| 9-10 | 问题建模:优化问题 + 难点(非凸/耦合)+ 分解思路 | 2min |
| 11-13 | 算法设计:总体框架图 + 各模块一页 + 复杂度一句话 | 2.5min |
| 14-17 | 仿真结果:一页一张图,"设置一句话 → 现象 → 结论"三段式 | 3.5min |
| 18 | 总结:贡献三点 + 局限与展望 | 1min |
| 19 | 致谢页("恳请各位老师批评指正") | 10s |
| 20 | 备份页:预计被追问的关键推导/补充数据(不讲,备查) | — |

**风格**:可参考往届优秀 PPT 的配色与版式(只借风格,内容必须全新);每页一个核心信息;文字能删则删,讲稿里说的不必都写在页面上。

**素材制备**(全部来自学生真实材料,不重画示意"假图"):

- 仿真图:从论文 PDF 按页裁剪(注意核对裁剪区域,裁错图是实际踩过的坑——裁完逐张 view 确认),或直接用 MATLAB 导出的 300dpi png;
- 公式:用 matplotlib mathtext 把 LaTeX 渲染成透明底 png 插入(PPT 里没有 MathType 时的稳妥方案);
- 系统架构图:用论文/开题报告里已有的图;
- **编号一致性**:PPT 上图的编号必须与论文最终版一致(PDF 顺序编号还是 docx 分组编号,先确认再做)。

## 2. 讲稿文档(三部分)

生成一份独立 docx(可用 njupt_docx_template.js):

1. **计时总表**:20 行(每页一行),列:页码 / 要点 / 计划用时 / 累计用时,总时长控制在 13~14 分钟(留缓冲,超时是答辩大忌);
2. **逐页逐字稿**:每页一段口语化但术语准确的完整讲词;涉及公式的页把 LaTeX 源码以等宽字体附在该页讲稿下,便于学生对着公式练习讲解;转场句写出来("接下来看算法如何求解这个问题");
3. **预设 Q&A(约 15 条)**:按五类各出 3 条左右,每条给 3~5 句的回答要点:
   - 建模类(为什么这样建模、假设是否合理、与现有模型区别);
   - 问题建模类(目标函数与约束的物理含义、为什么非凸、为什么这样分解);
   - 算法类(为什么选这个算法组合、收敛性、复杂度、参数怎么定);
   - 工程可行性类(实际系统怎么实现、硬件约束、与标准的关系)——**跨方向背景的评委常问这类**;
   - 仿真解读类(某条曲线为什么是这个走势、参数变了结果会怎样、基准方案公平吗)。

## 3. 评委背景调研(针对性准备)

拿到答辩分组名单后:

1. 对每位评委 web 搜索:学校教师主页、近年论文方向、专利、招生介绍(主页打不开就搜招聘公告/合作专利反推方向);
2. 输出每人"研究方向 → 可能从什么角度提问 → 准备哪几条 Q&A"的对照;
3. 典型规律:评委与课题同方向 → 备深挖类问题(推导细节、与 SOTA 对比);评委是相邻方向(如做硬件/器件的评通信算法课题)→ 备实现层面与基本概念的清晰解释,陈述时把跨方向的术语多铺垫一句。

## 4. 答辩资格审查与归档

- 答辩前确认 11 项材料齐全、题目各处一致、查重与 AIGC 达标(清单见 materials-checklist.md);
- 答辩后按答辩委员会意见做最后修改,更新论文终版 PDF,补齐指导记录末条与各表签字,按学院命名规则归档上交。

## 5. 演练建议(给学生的话术)

- 完整计时演练至少 3 遍,第 1 遍照稿、第 2 遍脱稿看页、第 3 遍模拟提问;
- 每页只记"这页我要说清的一件事";
- 被问到不会的:先复述问题确认理解,答自己确定的部分,不确定的坦诚"这一点目前工作没有覆盖,我的初步想法是…",不要硬编。
