# 南邮本科毕业论文格式规范(实测版)

以下规格逐项核对自一篇通过答辩的定稿论文。**学院当年下发的官方模板永远优先**;本文件的价值是在学生没有模板、或模板没写清的细节上,给出一套已被验收通过的完整答案。

## 1. 页面设置

- 纸张:A4(docx-js 中 width 11906 / height 16838 DXA);
- 页边距:接近 Word 默认(上下约 2.54 cm,左右约 3.17 cm),以官方模板为准;
- 前置部分(原创性声明、中英文摘要、目录):**无页眉、无页码**;
- 正文部分(第一章起,含结束语、致谢、参考文献):
  - 页眉:"南京邮电大学 20XX 届本科毕业设计(论文)",宋体五号居中,段落下边框加一条细横线;
  - 页脚:页码格式 `- N -`,宋体五号居中,从第一章第 1 页起算;
- 用 docx-js 实现时,前置与正文分成两个 section,正文 section 配 Header/Footer 并设 `pageNumbers: { start: 1 }`。

## 2. 论文部件与顺序

1. **封面**:学校统一格式。校名"南京邮电大学"+大字"毕 业 设 计(论 文)",下方表格依次为:题目 / 专业 / 学生姓名 / 班级学号(格式如"B20010101(B200101)",学号在前班级在后或按当年模板) / 指导教师 / 指导单位,底部"日期:20XX年X月X日至20XX年X月X日"(毕设起止日期,非提交日期)。封面一般直接在学校发的模板里填,不需要生成。
2. **原创性声明**:固定文字 + 作者签名 + 日期,学校模板自带。
3. **中文摘要**:标题"摘  要"(中间空两格)黑体居中;正文 2~3 段(背景引入 → 本文做了什么+方法 → 结果结论);末尾"关键词:"加粗顶格,词条间用中文分号,通常 4~6 个。
4. **英文摘要**:标题"ABSTRACT"加粗居中;内容与中文摘要逐段对应(不必逐句直译,但信息量一致);末尾"Key words:"加粗,词条与中文关键词一一对应,分号分隔。
5. **目录**:标题"目  录";列到三级标题;制表符前导符(点线)+ 右对齐页码。**必须用 Word 域自动生成**(前提:正文标题用了 Heading 样式),手敲的目录在最后改页码时会成为灾难。
6. **正文各章**:理工科典型结构为 5~6 章——第一章 概述(研究背景/国内外研究现状/研究内容/论文组织)、第二章 系统模型(或理论基础)、第三章 问题建模与分析(或方案设计)、第四章 算法/系统实现、第五章 仿真(实验)结果与分析、第六章 结论(总结+展望)。文科/管理类按"绪论—理论—分析—对策—结论"类推。
7. **结束语**:1~3 段**个人学习感悟**——这门课题让我学到了什么、过程中的体会、对方向的展望。注意它与第六章"结论"的区别:结论是技术性总结,结束语是第一人称的成长性总结,允许出现"我"。
8. **致谢**:感谢导师、师兄师姐、同学、家人,1~2 段,语气真诚具体,避免模板腔。
9. **参考文献**:见第 6 节。
10. **附录**(可选):核心代码、补充推导等。

## 3. 字体、字号与段落

| 元素 | 格式 |
|------|------|
| 正文 | 中文宋体、西文及数字 Times New Roman,小四(12pt),两端对齐,首行缩进 2 字符(12pt 字号下为 480 DXA),1.5 倍行距(`line: 360, lineRule: "auto"`;本届实测固定值 24 磅即 `line: 480, lineRule: "exact"` 与官方模板视觉一致,二者择一全文统一) |
| 章标题 | "第X章  标题"(章号与标题间空两格),黑体三号(16pt),居中,段前段后留空,每章另起一页 |
| 二级标题 | "2.1 标题"(编号后空一格),黑体四号(14pt),左对齐顶格 |
| 三级标题 | "2.1.1 标题",黑体小四(12pt),左对齐顶格 |
| 图题 | 图**下方**居中,"图5.1 系统性能收敛曲线"(编号"章.序",编号与题目间空一格),宋体五号(10.5pt) |
| 表题 | 表**上方**居中,"表5.1 仿真主要参数汇总",宋体五号 |
| 页眉 | 宋体五号居中 |
| 摘要/目录/结束语/致谢/参考文献的标题 | 黑体三号居中(与章标题同级) |

四级及以下层级不再编号,用"(1)(2)"或加粗短语引导。

## 4. 公式排版

- 公式独立成行、**水平居中**;编号格式"式(章-序)",如"式(2-5)",**右对齐**到右页边;
- 实现方式:同一段落内设两个制表位——居中制表位在版心中线、右对齐制表位在版心右缘;段落内容为 `[Tab]公式对象[Tab]式(2-5)`;
- 公式引导句以冒号结尾,如"用于解码用户 k 的信干噪比(SINR)为:";若引导句极短且其后紧跟公式(如"用户位置为:"),该引导句**顶格不缩进**;正常叙述段落仍首行缩进 2 字符;
- 公式后接续的说明用"其中,…为…"句式,该段顶格(承接公式)或按正常段落缩进,全文统一;
- 正文行内引用公式写"式(2-5)",不加"如下式"之类冗词;
- 所有公式用 MathType 制作(交付时给 LaTeX 源码,见 writing-workflows.md);行内变量(如 γ、K、单个符号)也应是 MathType 行内公式而非普通文本,以保证斜体/字体一致。

## 5. 图与表

- 图、表均居中;按章编号("图5.1""表5.1"),全文体系唯一(见下"编号一致性");
- 图先文后:正文先有"图5.1展示了……"的引述与分析段,图紧随其后;每张图必须在正文中被引用和解读,解读要写**真实曲线上发生的事**(走势、拐点、量级、对比关系),不写图上看不出来的结论;
- 多个强相关子图,优先**合并为一张图**:MATLAB 用 `tiledlayout`/`subplot` 排版,子图内标 (a)(b)(c),Word 中给一个总图题"图5.4 不同传输方案下的性能对比",正文可分别引用"图5.4(a)";这样避免"三张散图+下方大片空白"的版式问题(导师批注高发区);
- 图片以"嵌入型"插入(非浮动),前后用段前/段后距控制间距,不靠敲空行;
- 仿真图本身的风格规范(字体、线宽、配色)见 writing-workflows.md 第 9 节;
- 表用三线表或细实线表均可(从官方模板),表内文字五号,过宽的表可整体缩小字号。

### 编号一致性(高频翻车点)

docx 手工排版常出现"图5.1(a)(b)(c)"分组编号,而由 LaTeX/PDF 转来的版本是"图5.1、图5.2、图5.3"顺序编号。**全文(正文引述、图题、目录、答辩 PPT、讲稿)必须统一为一套**。改编号时全局搜索"图5"逐处核对,别只改图题。

## 6. 参考文献(GB/T 7714-2015)

文内引用:右上角上标 `[n]`,放句末标点前;同处多篇可写 `[9][10]` 或 `[6-8]`。

文末条目格式(与定稿论文逐字一致的两种高频类型):

- 期刊:`LIU F, CUI Y, MASOUROS C, et al. Integrated sensing and communications: Toward dual-functional wireless networks for 6G and beyond[J]. IEEE Journal on Selected Areas in Communications, 2022, 40(6): 1728-1767.`
- 会议:`DONG F, LIU F, LU S, et al. Secure ISAC transmission with random signaling[C]//Proceedings of IEEE Globecom Workshops (GC Wkshps). Kuala Lumpur: IEEE, 2023: 407-412.`
- 中文文献按 GB/T 7714 中文规则(作者全名,题名[J].刊名,年,卷(期):页码)。

要点:作者**姓全大写、名取首字母不加点**;3 人以上列前 3 人加", et al."(中文文献用"等");标题只首词与专名大写;[J]期刊/[C]会议/[D]学位论文/[M]专著;条目按文内首次出现顺序编号。

数量:导师普遍要求 **30 篇左右起**,近 5 年文献占比要高,中文文献适当搭配。**每一条都必须真实可查**:写入前用 web 搜索核实 DOI、卷期页码,与现有条目的大小写/标点/缩写风格逐项对齐;凑数或凭记忆写的文献在盲审/答辩时是硬伤。

## 7. 语言风格(中文学术语体)

- 客观陈述为主,正文用"本文",不用"我/我们"(结束语、致谢除外);
- 术语首次出现给全称+缩写:"移动边缘计算(Mobile Edge Computing, MEC)",此后统一用缩写;中英术语对照全文唯一(建议开写前先建术语表,见 writing-workflows.md);
- 句子信息密度要高但避免一逗到底;长短句交替;
- 数字与单位:数值和单位间留空格习惯按官方模板;变量斜体、单位正体、矩阵/向量粗体——这些由 MathType 保证,正文文本里不要出现裸的变量名;
- 每章末设"X.X 本章小结",**不少于 2~3 段**,把本章各节关键结论复述成"工作 → 结论 → 意义"的闭环(一句话的小结是导师批注高发区)。
